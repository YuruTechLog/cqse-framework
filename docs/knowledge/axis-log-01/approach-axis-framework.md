## 定義

Approach Axisは、人間の判断軸の「範囲」とAIのベクトル場の「範囲」が
近づき重なりうる領域を動的に探索・調整するための運用ツール。

核：「近づくという動作を操作化」し、成功と失敗を明確に定義して再現可能な判断プロセスを作る。

**起源（確定した一言）：**
「人間の軸にも範囲があって、論理和の部分を作ろうぜってかんじから？」
- 人間の軸 = 範囲（点ではない）
- AI = ベクトル場（場）
- Axis = 人間の軸の範囲 × AIのベクトル場の論理和（重なり領域）を扱う道具

**命名の理由（Approach確定）：**
- 動詞→操作できる・失敗を定義できる
- Approach Failure = ハルシネーション（状態遷移モデルへ自然に接続）
- 動的調整：近づく/離れるのダイナミクスを含む
- 一般認知されやすい

## 4層構造

| Layer | 名前 | 役割 |
|-------|------|------|
| Layer 1 | Axis Words | 操作を指示する言語語彙 |
| Layer 2 | Magnet Map | 各語彙の効力優先度 |
| Layer 3 | Model Profile | モデル別語彙効力差の参照表 |
| Layer 4 | Templates | 実行シーケンスの雛形 |

**運用原理：** Layer1で命令を作り→Layer2で優先度を決め→Layer3でモデル特性に合わせ→Layer4で実行。

---

## 操作プリミティブ（7種）

| プリミティブ | 操作内容 | コマンド例 |
|------------|---------|-----------|
| Position | AI出力を人の軸に対してどれだけ寄せるか | `setPosition(offset: +0.2)` |
| Magnitude | AI出力の重み付け | `setMagnitude(weight: 0.8)` |
| Constraint | 不可侵領域・禁止ルールの適用 | `applyConstraint(Ethics01, enforce)` |
| Filter | 出力フィルタ適用 | `applyFilter(factualOnly)` |
| Blend | 複数案の混合比率調整 | `blendOptions(0.7/0.3)` |
| Retry | 再アプローチ戦略と回数 | `retryApproach(maxAttempts: 2, strategy: tightenConstraints)` |
| Lock | 合意領域を固定して以降変更しない | `lockAxis(topic: summary)` |

## 状態モデル

```
Idle → Plan → Execute → Evaluate
                              ├─ Success（品質≥90 AND 人的レビュー合格）→ Lock → Handoff
                              ├─ Partial（70≤品質<90）→ 再アプローチ計画
                              └─ Failure（品質<70 OR ハルシネーション検出）→ 停止・原因分析・不可侵強化
```

**Approach Failure（ハルシネーション）の扱い：**
- 即停止
- 根拠トレース
- 不可侵領域の再強化
- 再設計

---

## 評価指標

| 指標 | 範囲 | 説明 |
|------|------|------|
| 品質スコア | 0–100 | 自動評価＋人的レビューで算出 |
| 信頼度 | 0–1 | モデル内部確信度 |
| 判定ラベル | 3種 | Success / Partial Success / Approach Failure |

---

## 実行フロー（短縮）

1. マッピング：人の軸レンジ × AIプロファイルを可視化
2. プランニング：段階（小/中/大）と操作コマンド列を決定
3. 実行：操作プリミティブを適用してAI出力を取得
4. 評価：品質スコア＋人的レビューで判定
5. 記録とHandoff：判定ラベルと次アクションをドキュメント化

## 失敗モードと対処

| 失敗モード | 原因 | 対処 |
|-----------|------|------|
| ハルシネーション | AIのベクトルが人の軸外へ逸脱 | 即停止→根拠トレース→不可侵領域強化→再設計 |
| 過剰適合 | 過度なプロンプト制約で多様性喪失 | 制約緩和→複数案取得→人的評価導入 |
| コスト超過 | 反復回数が多すぎる | 段階的閾値を厳格化→仕組み化検討 |
