# knowledge: tide-logs-01 / log02

## テーマ: OSSサンプルパッケージ設計

### 概要

免責付きデータ表示システムのOSS配布向けパッケージ構成と設計判断。

---

### パターン1: 免責選択式設計

**構造**:
- `config.json`にトーン識別子を持たせる
- `loader.min.js`がトーンに応じた免責画像をDOMに自動挿入
- 未設定時は警告 + データ展開停止

**適用場面**: 設置者に免責の自由度を持たせつつ、免責なし運用を防ぎたい場合

**トレードオフ**: 設置者の設定ミスでデータが表示されないリスクあり（設計として許容）

---

### パターン2: 多層免責構造

**構造**:
- UI（loader.min.js）
- data.json内のnoteフィールド
- README
- LICENSE追加条項

**適用場面**: OSSとしてフロントを公開し、設置者が免責を削除できる環境

**効果**: どこか一層削られても他層に残る

---

### パターン3: 作業単位分割（OSSリリース手順）

```
デザイン → 実装 → 難読化 → README整備 → テスト → リリース
```

**適用場面**: 個人・小規模チームのOSSリリース標準手順として転用可能

---

### 設計判断まとめ

| 課題 | 解決策 | 根拠 |
|------|--------|------|
| 免責削除リスク | 多層免責 | 削除耐性 |
| 設定忘れ対策 | 未設定時停止 | 構造的強制 |
| 法的強制不可 | LICENSE推奨+心理拘束 | OSSの限界 |
