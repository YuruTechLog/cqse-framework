# 第5章: 軸語彙リファレンス

**目的:** 軸語彙の5分類・磁力強度マップ・モデル別特性を参照可能な形で提供する。実務での使い方も示す。

## 骨子

- **5分類+補助（順序）の表を掲載（AXIS5軸確定版）**
  起点（Intent）/ 抽象度（Abstraction）/ 境界（Boundary）/ 優先順位（Priority）/ 方向性（Direction）/ 順序（Sequence・補助）
  各分類の役割・例語彙を一覧化。
  「順序が補助の理由」: AIは複数軸語彙を同時処理しにくい。順序語で段階的に刺すと軸同士の衝突を防げる。
  （参照: `axis-log-01/axis-words-magnet-map.md` §軸語彙5分類 / `axis-log-04/integrated_knowledge.md` §T01 §確定5軸）

- **磁力強度マップの掲載**
  効力が強い順: 方向性 > 境界 > 抽象度 > 起点 > 優先順位 > 順序
  （参照: `axis-log-01/axis-words-magnet-map.md` §磁力強度マップ）

- **「条件依存で変動」の注意書きの掲載**
  タスクの性質（抽象/具体）で優先度が入れ替わる。
  モデルのチューニング状態で特性が変わる。
  この順番は「傾向」であり固定ではない。
  （参照: `axis-log-01/axis-words-magnet-map.md` §磁力強度マップ §重要）

- **実務での使い方3点の掲載**
  1. 製造元を選ぶ基準として使う（概念設計→Anthropic / タスク完遂→OpenAI / 制約内→Google / コーディング→Microsoft）
  2. プロンプト設計時の優先語彙を製造元で変える
  3. 磁力強度マップを使った優先度設定（方向性先固定→境界で範囲限定→抽象度で粒度調整）
  （参照: `axis-log-01/axis-words-magnet-map.md` §実務での使い方）

- **合金比率（製造元の重みづけ思想）の説明**
  AI意味空間 = 合金の砂（鉄+アルミ混合）。軸語彙（磁石）を当てると鉄成分だけ引き寄せられる。
  製造元ごとに配合比が違う = 製造元別特性の原理。
  配合を決めるのはRLHF/DPO/Constitutional AIの設計思想。世代が変わっても製造元の思想は基本継承される。
  （参照: `glossary.md` §合金比率 / `drafts/model_comparison_axis.md` §軸2）

- **磁石サイズ（チャンク粒度）の説明**
  軸語彙の磁力は「何をひとつの塊として投げるか」で変わる。
  - 大きい塊 = 磁力範囲広い → 荒く引っかかる → 意図以外も巻き込む
  - 小さい塊 = 磁力範囲狭い → 意味の切れ目でぱっと切れる → 精度高いが引力弱い
  製造元別推奨サイズ: Anthropic=大きい塊でOK（原則非競合語彙前提）/ OpenAI=中粒度（起点明確）/ Google=細かく分割推奨（コンテキスト保持劣化リスクあり）
  論文対応: 表現学習（Representation Learning）/ Attention機構 = 磁力（コサイン類似度）
  （参照: `glossary.md` §磁石サイズ / `drafts/model_comparison_axis.md` §軸1）

## [参考メモ] 狙い撃ちの成否は合金×粒度で説明できる

| 状況 | 合金（製造元特性） | 粒度（チャンク） | 結果 |
|------|-----------------|---------------|------|
| 狙い撃ち成功 | 鉄分多い方向に軸語彙が当たってる | 磁石サイズが意味領域とマッチ | 引き寄せ成功 |
| 狙えない① | 鉄分が少ない方向に軸語彙を当ててる | — | 反応しない（製造元ミスマッチ） |
| 狙えない② | — | 粒度大きすぎ | 意図以外も巻き込む |
| 狙えない③ | — | 粒度小さすぎ | 引力弱くて届かない |

「なぜ狙い撃ちできるか」= 合金×粒度の掛け合わせ。失敗理由も同じ軸で説明できる。
