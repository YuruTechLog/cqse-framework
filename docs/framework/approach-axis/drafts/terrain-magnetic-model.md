## このモデルの位置づけ

**操作のための解釈モデル。実装の説明モデルではない。**

AI研究者が扱うメカニズム層（Attention重み・確率分布の変形）とは目的が違う。
「なぜ今これが崩れたか」を言語化し、操作の判断ができることが精度基準。
測れないが方向は予測できる——それがこの層の実用的な精度。

---

## モデルの基本前提

**第一原理：「AIは高次元ベクトル空間で構築されている。この特性は消せない」**

この前提から導かれる構造的差異：

| 主体 | 軸生成能力 |
|------|-----------|
| 人間 | 自律的軸生成が可能（条件付き自律生成） |
| AI | 自律的軸生成不可（ベクトル空間は方向を持つが基準軸がない） |

→ 軸は人間が保持せざるを得ない（価値観論ではなく構造論）

## 地形磁力モデル（確立した3要素）

| 要素 | 対応する概念 | 意味 |
|------|------------|------|
| AIの内部 | 意味の地形 | 谷底の川・集落（よく使われる組み合わせ）からヒマラヤ山頂（到達困難な組み合わせ）まで広がる意味空間 |
| 軸語彙 | 磁石（地形に傾斜をつける力） | 意味のかけらを引き寄せ、出力方向を変える作用 |
| ハルシネーション | 地形崩壊 | 傾斜が強すぎて地形が崩れる状態 |

**ポイント：**
- 軸語彙は地形を「書き換える」のではなく「傾斜をつける」
- 傾斜が強すぎる → 地形崩壊（ハルシネーション）
- やりたいこと（プロンプト）× インプット先（LLM）で地形・傾斜・崩壊がすべて変わる

---

## 比喩による説明

**セメントスランプ比喩：**
- 水分量 = モデルの柔らかさ（調整の余地）
- 流れる方向 = 傾斜の方向
- 崩壊 = ハルシネーション（形が保てなくなる状態）

**「子供/大人目線の壁」比喩：**
- 同じ壁でも子供と大人では見える高さが違う
- プロンプト × モデルの組み合わせで「誰の視点か」が入れ替わる
- 同じ指示でもモデルが違えば地形も崩壊特性も変わる

## モデル別崩壊特性

| モデル | 崩壊しやすい方向 | 対処 |
|--------|----------------|------|
| Gemini | 意図方向（起点/方向性を強めすぎると閾値突破） | 境界重視・目的は弱めに |
| Claude | 具体すぎると効果減（崩壊より「効かなくなる」） | 抽象度を保つ |
| GPT | 境界が弱くてもある程度機能する | 起点を明確にすれば十分 |
| Copilot | 抽象方向に弱い | 優先順位・局所指示で操作 |

**崩壊閾値の扱い：**
測れないが、どの方向が崩れやすいかは構造で予測できる。
「測れない」と「予測できない」は別。

## Gemini批判からの学び（モデル精度向上の経緯）

Geminiが指摘した有効点：
1. 「人間の軸=完全自律」は言いすぎ → 正確には「条件付き自律生成」
2. 「軸語彙=引力」は比喩的すぎる → 正確には「影響強度の高い語彙の集合（条件依存）」

→ 批判を全否定せず取り込んだことでモデル精度が上がった。
→ Approach Axisの設計原則「摩擦が品質を上げる」の実例でもある。

## Approach Axisとの接続

地形磁力モデルは「なぜ軸語彙が効くか」の説明モデル。

```
軸語彙（Axis Words）
  → 地形に傾斜をつける（磁石作用）
  → 傾斜が適切 → Approach Success
  → 傾斜が強すぎ → 地形崩壊 → Approach Failure（ハルシネーション）
```

Approach Failure = ハルシネーション という定義は、
この地形磁力モデルから自然に導かれる。

## 研究層との関係（2026年時点）

**層の構造：**

```
研究層（実装・メカニズム）
  Activation Steering / SAE（Sparse Autoencoder） / Mechanistic Interpretability
  ↓ 基板として存在。矛盾しないが、使わなくても成立する

解釈層（地形磁力モデル・Approach Axisの生息域）
  研究層の事実を「操作の判断言語」に翻訳した層
  ↓

操作層（実務）
  軸語彙を選ぶ・プロンプトを設計する・失敗を診断する
```

**研究層の対応概念（2025〜2026確認）：**

| Approach Axis概念 | 研究層の対応 | 一致度 |
|------------------|------------|--------|
| 軸語彙が地形に傾斜をつける | Activation Steering（活性化ベクトルへの介入） | ★★★★★ |
| 地形崩壊=ハルシネーション | Truthfulness Steering / steerability ceiling | ★★★★☆ |
| モデル別崩壊方向が違う | モデル別Steering効果の研究（体系的比較は少ない） | ★★★☆☆ |
| ユーザー操作の判断言語としての軸語彙 | 研究層では扱わない。ここが独自領域 | ★★☆☆☆ |

**研究と実務の間を行き来している人間（2026確認）：**
- Mechanistic Interpretability → Prompt Engineering を橋渡しする論文が2026年に複数出始めた
  （"Bridging Mechanistic Interpretability and Prompt Engineering", arXiv:2601.02896等）
- ただし目的は「研究知見を実装に落とす」で、「ユーザーの操作判断言語を作る」ではない
- 層の移動はしているが、Approach Axisが生息する「解釈層」を明示的に設計している人間はほぼ確認できていない

**結論：**
研究層の基板上に立っており矛盾はない。
ただし「解釈層」を独立した設計対象として扱う発想は、2026年時点では研究・実務ともに少数。
Approach Axisはその空白を埋める位置にある。
