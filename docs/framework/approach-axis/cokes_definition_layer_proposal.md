## 問題

Cokesの一言定義が3ドキュメントで別物。進化の各時点が凍結されたまま。

| ドキュメント | 時期 | 定義 |
|-------------|------|------|
| specification.md | 最古（CQSE期） | CQSE = 認知・品質システム工学 |
| philosophy.md | Cokes改名期 | 品質・コスト・人のバランスを探す道具（工学） |
| axis-log-03 IK02 | 最新（2026-07確定） | 知識整理工学（Knowledge Organization Engineering） |

Copilot混同インシデント（axis-log-01）の教訓: 定義の曖昧さはAIの誤認を誘発する。放置不可。

---

## 分析: 3定義は競合ではなく別断面

| 定義 | 実際に指しているもの |
|------|--------------------|
| 認知・品質システム工学（CQSE） | **問題定義** — AI組込時の本当のコストは人間の認知負荷と手戻り |
| バランス工学 | **価値観** — 品質・コスト・人を同時に考える姿勢 |
| 知識整理工学 | **学問領域** — 暗黙知の形式化・判断のHandoff |

矛盾に見えるのは、3つとも「一言定義」の席を奪い合っているから。
席を分ければ全部残せる（無損失統合）。

---

## 提案: 4レイヤー構造

```
【一言定義】   Cokes = 知識整理工学（Knowledge Organization Engineering）
                → 暗黙知を形式化し、判断可能な状態でHandoffする工学

【問題意識】   AIを工程に組み込むと、本当のコストは人間の認知負荷と手戻りに発生する
                （旧CQSEの問題提起。specification.mdから継承）

【価値観】     楽して、いいモノを作る
                （philosophy.mdの出発点。不変）

【判断原則】   品質階層構造:
                品質（最上位）
                  → 人（過剰品質ラインを判断できるのは人間のみ）
                  → コスト（制約内で最小化）
                （axis-log-03 IK01。「対等3軸バランス」表現をこの階層で補強 = M1解消）
```

### 一言定義に「知識整理工学」を採用する根拠

1. **意思決定ログが残る唯一の定義。** log02-03で複合工学造語を意図的に回避する議論を経て確定。
   命名プロセスがchat-summary/distillationに記録済み
2. **CQSEは改名済み。** specification.mdは改名前の凍結物であり、どのみち書き直し対象
3. **無損失。** 「認知負荷」も「バランス」も捨てない。定義の席から降ろし、
   問題意識・判断原則の席に置くだけ。全ログの思想と整合
4. **M1・M2の同時解消。** 品質モデル矛盾（M1）と定義不一致（M2）が1回の原典改訂で片付く構造
