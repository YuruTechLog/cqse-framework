# Lesson L002: 構造設計の判断は人間の軸

- 起因インシデント: なし（観察から）
- 発生日: 2026-07-12

---

## 気づき

`incidents/` と `lessons/` を分けるという構造判断は、人間が持ち込んだ軸。

AIは「記録する」はできる。
「インシデントと知見を分離して体系化する」という設計意図は持てない。

## なぜそうなるか

AIに「記録しておいて」と言えば、1ファイルにまとめて終わる。
「何のために残すか（再発防止・体系化・他AIへの引き継ぎ）」という目的軸がないと構造は生まれない。

目的軸を持てるのは人間だけ。構造はその軸から導かれる。

## Approach Axisとの接続

「軸は人間が持ち込む」の実例。
今回の軸: **「読みやすさのために責務を分離する」**
この軸があったから incidents/ と lessons/ が別フォルダになった。
軸がなければ1つのフォルダに混在していた。
