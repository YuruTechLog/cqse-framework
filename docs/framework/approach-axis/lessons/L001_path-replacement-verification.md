# Lesson L001: パス置換作業の検証手順

- 起因インシデント: `incidents/001_replace-all-path-collision.md`
- 適用対象: ファイル名変更・パス変更を伴う作業全般

---

## ルール

**パスを含む置換は `replace_all` 禁止。箇所ごと個別 Edit。**

理由: `replace_all` は文脈を持たない。「このパスは触らない」という意図を反映できない。

## 手順

1. 置換前に `grep` で対象箇所を全列挙
2. 箇所ごとに「変えていいか / 変えてはいけないか」を判断
3. 個別 Edit で1箇所ずつ修正
4. 修正後に `grep` で残存・誤置換を確認

## 判断基準: 変えてはいけないパス

- `docs/knowledge/` 配下（生ログ・蒸留物。framework作業でいじらない）
- `docs/chat-archives/` 配下（未加工ログ）
- 外部参照・出典として記録されているパス

## AIへの指示例（軸語彙）

```
`docs/knowledge/` 配下のパスは変更しない。
framework/approach-axis/ 配下のみ置換対象。
```

制約を明示しないと、AIはパターン一致で全置換する。
