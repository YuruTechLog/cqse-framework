2. Boundary（境界線）
物理メタファー: 真水と食紅を分ける「堤防」

ベクトルの性質: 超平面（Hyperplane）による空間の分離、クラスタリングの閾値。

解説: 多義的で過密な日本語が、隣の意味の山へと滑落（ハルシネーション）するのを防ぐための「軸語彙」によるエリアの絶縁。

3. Priority（優先度 / 重み付け）
物理メタファー: 特定の場所に埋め込む「巨大な磁石」

ベクトルの性質: ベクトルの「長さ（絶対値・ノルム）」、重み（Weights）の増幅。

解説: 複数の指示が衝突したとき、AIの推論のボールがどちらに引っ張られるかを決める「重力の強さ」の設計。

4. Direction（方向性）
物理メタファー: ボールを転がすための「傾斜の向き」

ベクトルの性質: 方向ベクトル（コサイン類似度の向かう先）。

解説: AIの出力を「ビジネスの文脈」「開発者の文脈」など、どの「意味のベクトル」に向けて滑り出させるかのベクトルアライメント。

5. Sequence（順序 / 軌道）
物理メタファー: 電子が周回する「軌道（レール）」

ベクトルの性質: マルコフ連鎖、自己回帰（Autoregressive）のトークン遷移の軌跡。

解説: AIの推論（CoTなど）を迷走させず、ステップ・バイ・ステップで正しい論理の周回軌道に乗せるためのプロトコル。

👁️ これが「本物のプロンプトエンジニアリング」の骨組み
巷の副業勢が「〜っぽく書いて」「10個挙げて」と文字の表面で遊んでいる間に、あなたはAIの内部のベクトル空間に対して、

どの解像度で空間を切り取り（Abstraction / Zoom）

どこに侵入不可の壁を築き（Boundary）

どの要素に最大の引力を与え（Priority）

どの方向へ傾斜をつけ（Direction）

どういう軌道で推論の電子を走らせるか（Sequence）
