🧩 ではどうする？
ここからは完全に実務向けの話。

✅ 1. AIに「作業文脈」を明示する仕組みを作る
AIに渡す情報は、
「文章」ではなく「状態」 にする。

例：

今：要件定義フェーズ

目的：粗い案でいいから方向性を出したい

人間の負荷：高い（疲れている）

AIの役割：選択肢を広げる

禁止事項：詳細化しすぎる、余計な枝葉を増やす

これを毎回渡すだけで、AIのズレは劇的に減る。

✅ 2. AIの出力を「人間の負荷基準」で評価する
普通は「正しさ」で評価するけど、
現場では “人間が楽になるかどうか” が基準。

評価軸はこれでいい：

読むのが楽か

判断が楽か

次の作業に進むのが楽か

手戻りが少ないか

認知負荷が低いか

これをAIに明示すると、出力が変わる。

✅ 3. AIの役割を「固定」しない
AIは万能じゃない。
だから役割を固定するとズレる。

代わりに、毎回こう指定する：

今は「要約者」

今は「選択肢生成者」

今は「批評者」

今は「仕様の穴を見つける役」

今は「人間の負荷を減らすための整理係」

役割を変えると、AIの軸が人間に合いやすくなる。

✅ 4. AIに「やりすぎ禁止」を明示する
AIは勝手に深掘りしたり、余計な提案をする。
これは現場の負荷を増やす。

だからこう書く：

深掘り禁止

余計な提案禁止

未確定情報の補完禁止

推測禁止

目的外の情報を出さない

これだけでズレが大幅に減る。

✅ 5. AIの出力を「人間の作業ステップ」に合わせる
AIは最終形を出しがちだけど、
現場では 途中のステップが欲しい。

例：

まず論点だけ

次に選択肢

次にメリット・デメリット

最後に結論

この順番で出すようにすると、負荷が激減する。

ログの活用方法
閾値判定：confidence < 0.5 または entropy > H_thr なら自動で HUMAN_REVIEW に回す。

キャリブレーション監視：予測確率と実際の正答率を定期集計し、信頼度の過信／過小評価を検出する。

差分分析：DIFF_METRICS を集計して、どのタスクで手戻りが多いかを特定する。

再学習トリガー：あるエラー理由が閾値超えで頻発したらモデル再学習をスケジュールする。

可視化：時間軸で confidence と KPI をプロットして品質低下の先行指標を探す。
