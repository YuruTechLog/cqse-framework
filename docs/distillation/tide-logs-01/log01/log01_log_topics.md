# log_topics: tide-logs-01 / log01

## テーマ一覧

### T-01: 免責表示設計
- 免責文を画像化（編集不可にするため）
- JSローダーがDOMに自動挿入
- config.json未設定時は警告+data.json展開停止
- トーン選択肢: 厳格 / 淡々 / 柔らか / 軽い

### T-02: JSローダー設計
- config.json読込 → 免責トーン確認 → 免責画像挿入 → data.json fetch → UI展開
- エラー時警告・最終更新時刻表示・スコア色付け・ラベル表示・まとめ文挿入を含む
- 免責とデータ展開をセットにする構造

### T-03: data.json構造確定
- 地点名 / 緯度経度 / 日別解析結果
- スコア / ラベル / まとめ文 / 理由文
- 潮位 / 風速 / 雲量 / 可視距離 / 最終更新時刻
- UIが扱いやすい粒度に固定済み

### T-04: OSS公開方針と免責削除抑止
- フロントをOSS公開、改造自由
- 軽難読化（変数名短縮・コメント削除・文字列軽エンコード）で免責削除の心理的ハードルを上げる
- ガチ難読化はOSSと相性悪いため不採用
- READMEに「免責は必ず設定」と記載（心理的拘束力）

### T-05: 実証試験のゆるさ設計
- 自動復旧なし
- 止まっても怒られない構造
- 期待値コントロールが最重要
- 「お仕事終わったあとで対応」が許容される前提

### T-06: 責任範囲の設計
- JSローダーを使う人のみ守る（免責が必ず表示される）
- URLを直接叩く層は自己責任（守る必要なし）
- data.jsonは解析結果をそのまま転がすだけ（法務リスクゼロ）
