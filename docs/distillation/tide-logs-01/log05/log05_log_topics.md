# log_topics: tide-logs-01 / log05

## テーマ一覧

### T-01: OSSフロント+非公開バックエンド構成の詳細設計
- フロントOSS公開、バックエンド（解析ロジック・APIキー）非公開
- data.jsonがフロント/バックエンドの境界インターフェース
- Cloudflare Pages経由で配信

### T-02: 気象業務法対応
- 「予報」ではなく「指標」として提供することで気象業務法の規制外に設定
- 免責表示が法的保護の要になる構造

### T-03: data.json肥大化問題と解決策
- 生気象データを含めると肥大化
- 解決策: 指標・スコア・ラベル等UIで使う値のみに絞る
- 中間計算値・デバッグ情報は含めない

### T-04: Slack通知による異常検知運用
- 定期実行が「飛ばない＝異常」として検知できる構造
- 自動復旧なし＋Slack通知で最小運用コストを実現

### T-05: 「自動復旧しない」宣言の有効性
- 期待値コントロールが最重要
- 「お仕事終わったあとで対応します」が許容される前提
- 宣言するだけで文句がほぼゼロになる心理効果

### T-06: 多層免責構造（削られても残る設計）
- UI / data.json / README / LICENSE / APIレスポンスの5層
- どこか一層削られても他層に残る
- OSSフロント公開時の防衛として最適

### T-07: data.jsonはオンメモリー → 免責の本丸はUI
- data.jsonはフロントのオンメモリーで消費されるだけ
- 一般ユーザーには見えない（devtools使用者のみ）
- 免責の本丸はUIに置くべきと確定

### T-08: 免責文「編集不可」の実現方法
- 画像化（編集不可・トーン別複数用意）
- バックエンドから免責文を読み込む構造（最強）
- ライセンス追加条項（推奨のみ・法的強制力なし）
- 署名（ハッシュ）付き画像（改造検知）

### T-09: 複数トーン免責画像セットの確定
- 「カラー」＝色ではなく「記事トーンの違い」
- 厳格 / 淡々 / 柔らか / 軽い の4種
- config.jsonでトーン選択 → JSが自動挿入
- 未選択時: 警告表示 + data.json展開停止

### T-10: 難読化レベルの確定
- 軽い難読化を採用（変数名短縮・コメント削除・文字列軽エンコード）
- ガチ難読化はOSSと相性悪いため不採用
- 免責バナー挿入ロジックのみ難読化対象
- フロント全体の難読化は不要

### T-11: 免責バナー挿入先のフォールバック
- `#disclaimer-area` が無ければ `document.body` 末尾に挿入
- 「やりすぎラインで止める」判断（Cokes的Humans保護）
