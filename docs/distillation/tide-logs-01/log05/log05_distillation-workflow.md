## 構造化結果

### ログの性質

設計深化ログ。log01〜04の方針を元に、OSSフロント公開に向けた防衛設計を詳細化。

### 設計の核心

「OSS公開しても免責が削られない構造」を実現するための多層防衛設計。

```
多層免責構造
    ├── UI（本丸 - 必ず表示）
    │     └── 免責画像（トーン別4種）をJSが自動挿入
    ├── data.json（補助 - ユーザー不可視）
    │     └── "note" + "auto_recovery: false"
    ├── README（OSS利用者向け）
    ├── LICENSE追加条項（心理的拘束力）
    └── APIレスポンス（fork後も残る最終防壁）

config.json
    ├── disclaimer: "soft" 等（トーン選択）
    └── null/未設定 → 警告 + data.json展開停止

難読化レベル: 軽い（免責ロジックのみ）
```

### 判断の足跡

| 判断 | 選択 | 理由 |
|------|------|------|
| 免責の本丸 | UI | data.jsonはオンメモリー・ユーザー不可視 |
| 「カラー」の意味 | トーン（記事調子） | 色ではなくstyle差異 |
| 難読化 | 軽い難読化のみ | OSSと相性悪いため全体難読化不採用 |
| バナー設置領域なし | body末尾フォールバック | やりすぎ判断で最小コード追加 |
| 過剰防衛 | 停止 | Cokes的Humans保護（運用者の負荷最小化） |

## 再利用ポイント

- 多層免責構造（5層）はOSS公開プロダクト全般に転用可能
- 「免責未選択=展開停止」パターンはゲートとして汎用転用可能
- 「やりすぎで止める」判断の言語化がCokesのHumansルール実践例
- 気象業務法対応（「指標」として免責で守る）は類似法規制プロダクトに応用可能
