# log_topics: tide-logs-01 / log06

## テーマ一覧

### T-01: AIと人間の役割分担モデル
- AI: チェックポイント候補生成・分岐分類・状態解析（広く見る）
- 人間: 目的設定・重要分岐の選択・最終判断（狭く深く見る）
- 役割の線引きは「候補生成/選択」の境界

### T-02: イベント駆動AIエージェント設計
- 外部イベント（メール受信・PR作成等）をトリガーに自発性なしで動く
- 予定イベントポイントを事前設計することで「未来を読んだように」振る舞える
- CI/CDパイプライン・GitHub Actionsと構造的に一致

### T-03: 「自走っぽさ」の正体
- 実態: 状態遷移図・目的・イベントポイント（外付け構造）を辿っているだけ
- 外から見ると自律的に見える
- Codex・現行AIエージェントはすでにこの構造を実装済み

### T-04: 状態保存の設計判断
- Notion型（ポーリング監視）→ イベント駆動に不向き・非効率
- RAG/DB型（外付け状態保持）→ イベント駆動に適合
- 状態はAI内部ではなく外付けで管理が正解

### T-05: OSSフロント+非公開バックエンド構成との接続
- フロントOSS＝透明性確保
- バックエンド非公開＝効率化・ロジック保護
- この構造がイベント駆動エージェント設計と整合

### T-06: メタ的気づき
- 「遊びながら構造を組み上げる」ことが設計者の強み
- 個人用途でオーバースペックなパイプラインは捨てる判断が正しい
- 本議論が自然に「即戦力となる仕様書レベル」に達した
